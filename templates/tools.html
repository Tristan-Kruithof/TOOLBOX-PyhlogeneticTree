Tools

Here is some brief information about the tools used for our website. These include:

  • Biopython
  • MEGA cc
  • ETE4

Biopython is used to handle our user input and request taxonomy data from the NCBI database. MEGA cc is used to align DNA sequences, compare them with each other, and create a phylogenetic tree based on this comparison. ETE4 is used to make and compare trees with a newick file as input.

Biopython

Biopython is a tool designed for the language python. It can be used to retrieve information from NCBI. The user can specify the database, then add a search term. Biopython will then return all the results from the search term in the chosen database.

We use biopython to retrieve a specific gene in the mitochondrial DNA of species. When you fill in the name of an organism on this website, biopython will search NCBI for the gene sequence of that organism. All the organisms that you want to include in your phylogenetic tree will have this particular gene sequence and this sequence is used to compare the organisms.

Mega CC

Mega cc is used to align DNA sequences, compare them with each other, and create a phylogenetic tree based on this. In our case, it creates a file in newick format that contains data for a phylogenetic tree, which can then be read by ETE4.

It is very important to align the DNA sequences before comparing them, because this ensures that you are comparing homologous positions. If you dont align them properly, your tree will be made based of incorrect data and just be completely off. A homologous position means that, from an evolutionary perspective, they come from the same area, so I am comparing the same genetic data from different sequences. If you do not align them, or do not align them properly, you may not be comparing the same genetic data, which can lead to errors in the phylogenetic tree.

ETE4

ETE4 is used to create and visualize phylogenetic trees. We use visualization because MEGA cc already creates the newick file that we import into ETE4. ETE4 can retrieve taxonomy data from organisms, such as their kingdom, domain, class, order, etc. You can then visualize this data in ETE4 by rendering an image of the tree or by creating an interactive visualization on a web page using the built-in functions, smart-view, and explorer.

You can view the different nodes (junctions) of the tree and retrieve mutations with them. A phylogenetic tree shows the evolutionary relationships between species. If the branch lengths are based on genetic distance, you can also derive information about relative mutation rates.

Input and Output

Each tool requires its own specific input. For the user, they will only have to provide the start input for Biopython and receive the output of ETE4, which is the phylogenetic. The rest of the inputs and outputs will be tuned to each other and not be seen by the user, but for completness they will also be explained here.

In order to start the whole process biopython requires the input of the user in order to process it for mega. To make sure that people off all levels of biological knowledge can use this site, there will be a lot of options for input, which include: the common name of species, the scientific name of species, fasta files of the correct gene or even multi fasta files of the correct gene. The chosen gene per kingdom is already set, but can be changed by the user if they want to. Biopython then takes this data and puts all the dna sequences into a multifasta file, if it isnt already and passes it on to Mega.

Mega needs two inputs: an input file with the dna sequences and a settings file, so it knows which methods to use when analyzing. The input file is provided by biopython and the settings file is already defined by the website, with as for now know plans to let users edit it. When Mega has processed this it will spit out an newick file, which can be used by ETE4 to create a phylogenetic tree.

ETE4 has plenty of design options, of which a few such as changing colours, sizes, and even adding pictures for each organism will be added to the site.
